# 保存会话

## 目标

把注册/登录后的 Tabbit session 保存到本地安全存储，并把账号加入 AccountPool。当前 foundation 使用 `secretStore.writeSecret(ref, value)` 保存 raw cookie/session，再把 `cookieJarRef` 和普通账号字段写入 account store。

## 保存内容

| 内容 | 存储方式 | 说明 |
|---|---|---|
| cookie jar / cookie header / session token | `secretStore`，默认 ref 形如 `secrets/<accountId>.cookie` | 必须脱敏日志；不进入账号 JSON |
| userId | 普通字段或部分脱敏 | 用于排错和状态关联 |
| email | 普通字段但日志脱敏 | 账号标识 |
| accessTier | 普通字段 | free/pro/unknown |
| quotaState | 普通字段 | 维护任务刷新 |

## 原子性

- cookie/session 写入成功后，才把账号状态设为 active。
- 写入中断时状态保留为 provisioning，允许 resume。
- `importSession()` 也遵循 secret-first：secret 写入失败时不持久化 active 账号。
- 账号 store 即使被传入 raw `cookieHeader`、`cookie`、`session`、`token` 字段，AccountProvisioner 也会在保存前剥离。

## 当前可接受的 session 字段

`extractSessionSecret(result)` 会按顺序读取：

1. `cookieHeader`
2. `cookieJar`，对象或数组会 JSON.stringify
3. `cookie`
4. `session`
5. `sessionToken`
6. `token`

全部缺失时返回 `null`。注册/登录成功响应缺少 session material 时，`createAccount()` 将账号标记为 `suspect`；导入账号缺少 session material 时，`importSession()` 返回 failed action 且不写账号。

## 脱敏

- cookie value 只显示前后 3 个字符或 hash。
- 邮箱 localPart 默认显示前 2 位 + hash。
- 验证码不写日志。

- [x] 写入失败不产生 active 账号。
- [x] `createAccount()` 缺少 session material 时标记 `suspect`。
