# 模型目录同步

## 目标

把 Tabbit Web 模型目录转换为网关可用的模型能力表，并保留 `tabbit/priority` 统一入口。

## 已知线索

- 模型目录接口：`GET https://web.tabbit.ai/proxy/v1/model_config/models?a=0&scene=chat`。
- 返回结构为 `{ "models": [...], "status": ... }`；模型项使用 `display_name` 作为真实选择名。
- 现有 Tabbit2API 已在 `/v1/models` 中暴露 OpenAI/Anthropic 风格模型列表。
- `tabbit/priority` 是公开推荐的虚拟模型别名，不能随意移除或改名。

当前已验证：

- `Default` 的 `model_access_type` 为 `free_unlimited`，可作为 `tabbit/priority` 的实际发送模型。
- `GPT-5.5` 的 `model_access_type` 为 `premium_only`；普通账号发送会返回模型权限错误 fixture，不应作为默认成功探针模型。
- `supports_images` / `supports_tools` 在真实目录中为布尔字段，归一化时分别输出为 `supports_images` / `supports_tools`。

## 输出字段

| 字段 | 说明 |
|---|---|
| id | 网关模型 ID，例如 `tabbit/priority` 或 `tabbit/<name>` |
| selectedModel | 实际 Tabbit 模型名 |
| displayName | 面向日志和调试的展示名 |
| supports_tools | 是否支持工具调用 |
| supports_images | 是否支持图片输入 |
| model_access_type | 真实访问类型，例如 `free_unlimited`、`free_metered`、`premium_only` 或 `unknown` |
| available_in_tabbit_catalog | 是否来自当前 Tabbit 目录 |

## 缓存策略

- 当前默认缓存 5 分钟，`listModels({ force:true })` 可强制刷新。
- 手动 probe 或账号权益变化后应允许强制刷新。
- 目录获取失败时当前会返回协议错误；后续如接入持久缓存，必须在健康状态中暴露 stale 标记。

## 边界条件

- 不同账号权益可能看到不同模型目录；当前 `listModels()` 不带账号维度缓存。
- 模型支持工具/图片的字段可能缺失，需要默认 `false` 或 `unknown`。
- 目录接口失败不能导致 `/health` 失败；但 `/v1/models` 应返回兼容错误或缓存。

## 测试

- 真实 `display_name/model_access_type/supports_images/supports_tools` 目录 fixture 到 OpenAI models shape。
- `tabbit/priority` 总是存在。
- 目录为空、字段缺失、重复模型 ID 的归一化测试。
