# M07-配置密钥

## 定位

配置与密钥模块负责读取环境变量、本地状态路径、账号 session 存储引用和日志脱敏策略。它必须先于协议客户端和账号池实现。

| `TABBIT_POOL_PROTOCOL_MODEL_CATALOG_PATH` | 模型目录路径覆盖 | `ProtocolTabbitClient` 默认值 | 否 |

协议相关变量是显式 opt-in。默认 `protocol.enabled=false`，`accounts probe`、`probe protocol`、`maintain` 和 gateway `/v1/models` 不会触发真实 Tabbit 网络；设置 `TABBIT_POOL_PROTOCOL_ENABLED=true` 或任一协议 endpoint path 后，CLI/gateway 才会创建 `ProtocolTabbitClient`。其中 `TABBIT_POOL_PROTOCOL_MODEL_CATALOG_PATH` 只影响模型目录读取：显式配置后，gateway 默认 `/v1/models` provider 会调用 `ProtocolTabbitClient.listModels()`，默认携带 `a=0&scene=chat`；未配置且未注入 provider 时仍只返回 `tabbit/priority`。`TABBIT_POOL_PROTOCOL_SESSION_VERIFY_PATH=/api/v0/user/base-info` 是已校准的真实账号/session 检查路径，用于 `accounts probe` 和 `probe protocol --operation verifySession`。`TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH=/api/commerce/quota/v1/usage` 是已校准的真实额度 usage 查询路径，用于 `probe protocol --operation refreshQuota` 和 `tabbit-pool maintain` 中的 `refreshQuota`；请求依赖本地 secret store 中的 Cookie 和账号 `userId`，不生成 `x-signature` / `x-nonce`。`TABBIT_POOL_PROTOCOL_SIGN_IN_PATH=/api/commerce/activity/v1/sign-in` 会让默认 `maintain` 执行已验证每日签到；若同时配置 `TABBIT_POOL_PROTOCOL_SIGN_IN_STATUS_PATH`，维护会先查 `signedToday`。activity/newbie/placement/reward/lottery 五个 path 只启用只读 commerce 状态/资源探针；其它 M05 side-effect path 只启用显式 protocol probe / gateway protocol client factory，POST 型方法还要求 `confirmSideEffect:true`。这些 path 不会让 `maintain` 自动领取活动权益、抽奖或使用重置券。`TABBIT_POOL_PROTOCOL_CHAT_SESSION_ID` 是真实发送接口的必需运行时上下文，不应写入公开示例中的真实值。`TABBIT_POOL_PROTOCOL_ATTACHMENT_UPLOAD_PATH=/proxy/v0/cos/presigned-upload-url` 单独配置时仍兼容旧的显式上传骨架；同时配置 `TABBIT_POOL_PROTOCOL_ATTACHMENT_COMPLETE_UPLOAD_PATH=/api/v0/cos/complete-upload` 后，`uploadAttachment()` 会走 presign -> COS PUT -> complete-upload，真实 `/api/v1/chat/completion` 分支也会在发送前把 raw/base64 附件自动上传并映射为 `references[].metadata.file_id`。

`TABBIT_POOL_COMPAT_STRIP_CLIENT_TOOLS` 默认关闭。它只用于 Codex / Claude Code 本地文本链路验收这类“客户端可能自动附带内置协作工具，但本轮不要求模型调用工具”的场景；普通 API 工具定义仍应保持默认透传，真实 `/api/v1/chat/completion` 原生工具字段已确认不支持，不能宣称已支持上游原生工具调用。

`TABBIT_POOL_TOOL_LOOP_MODE` 控制网关工具 loop 策略。默认 `client_executes_tools_first` 表示保持官方 API 兼容结构，由 Codex、Claude Code 或普通 SDK 调用方执行工具后回传 tool result；`disabled` 表示剥离工具字段后走普通文本链路；`local_executes_tools` 表示由 LocalToolLoopRunner 代执行工具，但只有 gateway 宿主注入 `executeLocalToolUse` 或 `localToolExecutor.execute` 时才会真正执行。网关默认不内置 shell、web、js、fetch 等工具；缺少 executor 会返回明确 invalid_request，避免误以为本地工具执行已自动可用。

真实 Tabbit 域名在当前环境需要本机代理；Node 直连探针应显式使用：

~~~powershell
$env:NODE_USE_ENV_PROXY="1"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
$env:HTTP_PROXY="http://127.0.0.1:7890"
~~~

这些是本机网络环境变量，不会被 `loadConfig()` 读取为项目配置，也不应写入账号元数据或 fixture。

## 状态目录草案

~~~text
<state-dir>/
  accounts.json             # 已实现：账号元数据 JSON store
  accounts.sqlite           # 可选替代：后续 SQLite store
  secrets/                  # 已实现：文件型 secret 引用存储，raw secret 不写入 accounts.json
  fixtures/                 # 本地抓包样本，默认不提交
  logs/                     # 脱敏日志
  protocol-cache.json       # sign key / model catalog cache
~~~

## 脱敏规则

- `AC-` 开头 API key 只显示 `AC-***`。
- cookie/token 只显示 hash。
- 邮箱 localPart 默认显示前两位和 hash。
- 验证码永不输出。

## 配置加载顺序

1. 内置默认值。
2. 配置文件。
3. 环境变量。
4. CLI 参数。

## 测试

- 默认监听本地地址。
- env 覆盖生效。
- API key 和 cookie 脱敏。
- 缺少 `YYDS_MAIL_API_KEY` 时只禁用邮箱/注册功能，不影响已有账号请求。
- 缺少 `TABBIT_POOL_PROTOCOL_*` opt-in 时默认不创建真实 protocol probe client，不猜测未知 Tabbit endpoint。
- `TABBIT_POOL_PROTOCOL_MODEL_CATALOG_SCENE`、`TABBIT_POOL_PROTOCOL_REQ_CTX` 和 `TABBIT_POOL_PROTOCOL_CHAT_SESSION_ID` 会进入 `config.protocol` 并传给默认协议客户端。
- `TABBIT_POOL_PROTOCOL_ATTACHMENT_UPLOAD_PATH` 会被解析到 `config.protocol.attachmentUploadPath`，并单独触发 `protocol.enabled=true`。
- `TABBIT_POOL_PROTOCOL_ATTACHMENT_COMPLETE_UPLOAD_PATH` 会被解析到 `config.protocol.attachmentCompleteUploadPath`，并单独触发 `protocol.enabled=true`；与 upload path 同时配置时启用真实 COS 三步上传和 send 前 raw/base64 自动上传。
- `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH` 会被解析到 `config.protocol.quotaUsagePath`，并单独触发 `protocol.enabled=true`；真实推荐值是 `/api/commerce/quota/v1/usage`。
- `TABBIT_POOL_PROTOCOL_ACTIVITY_LOTTERY_PATH`、`TABBIT_POOL_PROTOCOL_NEWBIE_EXPLORATION_PATH`、`TABBIT_POOL_PROTOCOL_PLACEMENT_RESOURCES_PATH`、`TABBIT_POOL_PROTOCOL_REWARD_CARD_RECORDS_PATH` 和 `TABBIT_POOL_PROTOCOL_LOTTERY_HIT_RECORDS_PATH` 会分别进入 `config.protocol` 并单独触发 `protocol.enabled=true`；真实推荐值分别是 `/api/commerce/activity/v1/lottery/me`、`/api/commerce/activity/v1/newbie-exploration/me`、`/api/commerce/placement/v1/resources`、`/api/commerce/reward/v1/card-records`、`/api/commerce/lottery/v1/hit-records`。
- `TABBIT_POOL_PROTOCOL_SESSION_VERIFY_PATH` 会被解析到 `config.protocol.sessionVerifyPath`；真实推荐值是 `/api/v0/user/base-info`，默认 method 为 `GET`。
- `TABBIT_POOL_COMPAT_STRIP_CLIENT_TOOLS` 会进入 `config.compat.stripClientTools`，默认 false，非法布尔值会报错。
- `TABBIT_POOL_TOOL_LOOP_MODE` 会进入 `config.compat.toolLoopMode`，默认 `client_executes_tools_first`，支持 `disabled` 和 `local_executes_tools`，非法模式会报错。
- `JsonAccountStore` 会剥离直接 cookie/session/token/password/apiKey 字段，保留 `cookieJarRef` 这类引用。
- `FileSecretStore` 会把 `cookieJarRef` 解析到 stateDir 内，并由 gateway 在运行时 hydrate 到临时账号对象。
