# 重置券使用

## 目标

当账号额度耗尽且存在重置券时，自动使用重置券恢复额度。

## 触发条件

- 账号状态为 `quota_exhausted`。
- `resetCouponCount > 0`。
- 账号未处于登录失效或风控状态。
- 未超过每日自动使用上限。

## 策略

- 默认只在请求需要时使用，避免提前消耗。
- 使用前后都刷新额度。
- 失败时记录 action log，不重复快速重试。

## 待还原点

- 使用后额度刷新是否同步完成。

## 已校准只读记录

当前已还原的奖励卡和抽奖命中记录都是只读查询，不会使用重置券：

~~~text
GET /api/commerce/reward/v1/card-records?user_id=<user_id>&offset=0&limit=10&order.field=award_time&order.order=desc
GET /api/commerce/lottery/v1/hit-records?user_id=<user_id>&offset=0&limit=20
~~~

两条 GET 都只依赖已登录 Cookie、`accept: application/json`、`unique-uuid` 和可选 `x-req-ctx`，不使用 `x-signature` / `x-nonce`。`listRewardCardRecords()` 可附加 `rewardPackageId` 和 `awardStatus`，`listLotteryHitRecords()` 可附加 `mainPoolId`；二者都要求账号或输入中有 `userId`，缺失时不触网。`unused_reset_coupon_count` 仍来自已校准的 quota usage 响应，并归一化为账号 `resetCouponCount`。

## 已校准重置券相关接口

真实重置券列表和 reset-coupon activity participate 已还原为显式 probe：

~~~text
GET  /api/commerce/benefit/v1/coupon/list?user_id=<user_id>&coupon_type=weekly_reset_coupon&offset=0&limit=50
POST /api/commerce/activity/v1/participate
GET  /api/commerce/product/v1/sku/usage-reset-coupon
~~~

重置券列表使用已登录 Cookie、`unique-uuid` 和可选 `x-req-ctx`，不使用 `x-signature` / `x-nonce`。reset-coupon activity participate 的 body 为：

~~~json
{
  "user_id": "<user_id>",
  "request_no": "<caller-generated>"
}
~~~

该 POST 使用 `x-req-ctx`、`unique-uuid` 和 `Content-Type: application/json`。当前 live probe 证明 fake user_id 也可能返回 200，因此不能把 200 直接解释为真实领取成功，也不能把该 activity participate 等同于“消耗已有重置券”。默认 `useResetCoupon()` 仍只依赖注入协议操作，不会因配置 path 自动用券。

`request_no` 由调用方生成，生产调用点使用短字符串形态；`ProtocolTabbitClient.participateResetCouponActivity()` 和 CLI schema 都限制为非空且不超过 64 字符。响应会把 `activity_id` 和 `participation_result` 归一化为 `activityId` 和 `participationResult`。2026-07-03 live 脱敏 evidence `output/tabbit-benefits-side-effects/side-effect-live-20260703T082435232Z.sanitized.json` 显示真实账号调用返回 `{ "activity_id": 10001, "participation_result": "already_participated" }`，但重置券列表前后 hash 一致，quota usage 前后 hash 一致，不能作为“消耗已有重置券恢复额度”的成功证据。`getUsageResetCouponSku()` 当前环境可返回 404 `PRODUCT_NOT_PURCHASABLE`。

## 测试

- [x] 无券或账号不是 `quota_exhausted` 时 skipped。
- [x] 使用成功后账号转 active，`resetCouponCount` 扣减，并可写入刷新后的 quotaState。
- [x] 使用失败保持原状态并返回 failed action。
- [x] 明确 `rate_limited` / `network_error` / `upstream_error` 维护错误会把账号转入 `cooldown`，保留重置券数量，并写入脱敏 `lastError` 与 `cooldownUntil`。
- [x] 真实奖励卡/抽奖命中记录只读查询已校准为 probe operation。
- [x] 真实重置券列表和 reset-coupon activity participate path/header/body 已校准为显式 probe。
- [x] reset-coupon activity participate 响应归一化 `activityId` 和 `participationResult`，但不把 200 自动解释为用券成功。
- [ ] 真实“消耗已有重置券恢复额度”的成功语义和后置额度刷新仍待安全 evidence 后接入。
