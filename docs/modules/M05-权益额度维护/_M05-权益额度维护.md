# M05-权益额度维护

## 定位

BenefitsMaintainer 负责让账号池长期可用：签到、领取活动 Pro、查询额度、使用重置券，并把结果写回 Account 元数据。当前已实现离线可测的编排层；真实 quota usage 查询已校准为 `GET /api/commerce/quota/v1/usage?user_id=...` 并通过显式 `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH` opt-in。每日签到状态/POST 已有成功闭环证据，显式配置 `TABBIT_POOL_PROTOCOL_SIGN_IN_PATH` 后默认维护链可执行 `dailyCheckin`。activity lottery、newbie exploration、placement resources、reward card records 和 lottery hit records 已校准为只读状态/资源探针；权益券列表、真实重置券消耗、activity participate、usage reset coupon sku、抽奖机会/奖池/记录/POST 已校准为显式 `probe protocol` operation。活动 Pro 和抽奖 POST 型副作用仍只允许显式 probe 或调用方注入实现；真实用券消耗已通过 `useResetCoupon` 显式 probe 闭环，但默认维护链仍等待安全选券和后置额度刷新策略，不会因配置 path 自动执行。

## 公开能力

| 能力 | 方法草案 |
|---|---|
| 单账号维护 | `maintainAccount(account)` | 已实现，固定顺序执行 refresh quota、claim Pro、daily check-in、use reset coupon，并返回 actions。 |
| 批量维护 | `maintainAllAccounts(accounts?)` | 已实现；可维护传入账号数组，或在构造时绑定 `accountStore` 后自动 load/save。 |
| 查询额度 | `refreshQuota(account)` | 已实现编排与状态更新；显式配置 quota usage path 时可由 `ProtocolTabbitClient.refreshQuota()` 查询真实用量。 |
| 查询权益状态/资源 | `ProtocolTabbitClient` 只读探针 | 已校准 lottery/newbie/placement/reward/lottery records 五类 GET，可由 `probe protocol` 触发；不进入默认维护动作链。 |
| M05 显式协议探针 | `ProtocolTabbitClient` side-effect probe | 已校准 sign-in、coupon、activity participate 和 lottery 相关 GET/POST path；除已接入维护的每日签到外，POST 必须显式确认且默认不接入维护动作链。 |
| M05 evidence 审计 | `fixtures audit --scope benefits` | 只读统计每日签到、活动 Pro 成功、真实重置券消耗和抽奖成功 fixture 覆盖；不执行副作用，不改变默认 readiness。 |
| 每日签到 | `dailyCheckin(account)` | 已实现同 UTC 日期跳过与成功写回；显式配置 `TABBIT_POOL_PROTOCOL_SIGN_IN_PATH` 后默认 CLI maintainer 会注入真实签到，配置 status path 时先查 `signedToday`。 |
| 领取 Pro | `claimProIfAvailable(account)` | 已实现 paid tier/proClaimed 跳过与成功写回，依赖注入的 `protocolClient.claimProIfAvailable`。 |
| 使用重置券 | `useResetCoupon(account)` | 已实现 eligibility 判断、成功转 active 与券数扣减，依赖注入的 `protocolClient.useResetCoupon`。 |

## 当前接口

~~~ts
const maintainer = new BenefitsMaintainer({
  accountStore, // 可选；用于 maintainAllAccounts() 自动 load/save
  protocolClient: {
    refreshQuota(account) {},
    dailyCheckin(account) {},
    claimProIfAvailable(account) {},
    useResetCoupon(account) {},
  },
  now: () => new Date(),
});
~~~

缺少某个协议方法时，对应动作返回 `skipped`，不会猜测真实 endpoint。协议方法抛错时，对应动作返回 `failed`，并继续执行后续维护动作。错误消息会经过脱敏。若错误带有明确 `category`，维护层会更新账号状态：`login_required` -> `login_expired`，`rate_limited` / `network_error` / `upstream_error` -> `cooldown`，`protocol_changed` / `forbidden` -> `suspect`，`quota_exhausted` -> `quota_exhausted`；未分类错误仍保持原账号状态。`maintainAllAccounts(accounts?)` 会按账号顺序调用 `maintainAccount()`，返回 `{ accounts, changed, results }`；无参数调用时需要构造时传入 `accountStore`，并且只有任一账号 `changed:true` 时才调用 `saveAccounts()`。CLI 默认仍不触网；配置 `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH` 后，默认 maintainer 会从本地 secret store hydrate cookie 并执行真实 `refreshQuota`；配置 `TABBIT_POOL_PROTOCOL_SIGN_IN_PATH` 后，会执行已验证 `dailyCheckin`。配置只读 commerce path 只影响 `probe protocol` / gateway protocol client factory；活动 Pro、抽奖和真实用券不会自动执行。

`tabbit-pool fixtures audit --scope benefits --json` 可用于读取本地脱敏 protocol probe fixture 并报告 M05 副作用 evidence 缺口。当前 `dailySignIn` 成功和 `useResetCoupon` 真实用券成功都可计为 ready；活动 Pro 成功和抽奖成功必须等安全成功 fixture 后才会 ready。`participateResetCouponActivity` 只作为活动参与 evidence，永不计为真实用券成功；`drawLottery` 也必须有 draw/lottery 专属成功或非空奖品/命中记录。`readiness doctor` 会为真实用券缺口给出 `useResetCoupon` template/validate/confirm_validate/probe 命令，并要求 `TABBIT_POOL_PROTOCOL_BENEFIT_COUPON_USE_PATH` 前置配置；离线 `consumeResetCoupon` 模板仍可用于导入外部脱敏 endpoint/body/result hash evidence，但 `probe protocol --operation consumeResetCoupon` 会被拒绝。该 scope 只读取和统计 M05 side-effect 白名单 operation，避免聊天、auth 或 session fixture 污染权益缺口判断。

## 调度建议

- 启动后延迟执行一次轻量状态刷新。
- 每日固定窗口执行签到和权益领取。
- 请求失败出现 quota_exhausted 时触发单账号维护。
- 所有维护任务都要有并发锁，避免同账号重复领取。

## 完成标准

- [x] 维护动作以 `actions` 返回，可审计。
- [x] 单个动作失败不会阻断同账号后续动作。
- [x] quota_exhausted 账号在额度恢复或重置券成功后可回到 active。
- [x] 批量维护账号数组或绑定的 `accountStore`，且无变化时不写盘。
- [x] 真实 quota usage 查询接入。
- [x] 真实只读 commerce 状态/资源探针接入：lottery/me、newbie-exploration/me、placement resources、reward card records、lottery hit records。
- [x] 真实每日签到状态/POST、重置券列表/activity participate、抽奖机会/奖池/记录/POST path 接入显式 protocol probe。
- [x] M05 side-effect fixture 覆盖可通过 `fixtures audit --scope benefits` 只读审计。
- [x] 真实“消耗已有重置券”的 endpoint/body/result 成功语义已通过 `useResetCoupon` 显式 probe 和 benefits audit 闭环。
- [ ] 活动 Pro 成功领取 body、抽奖成功响应、用券后的额度刷新和安全自动选券策略仍待安全 evidence 后接入维护动作。
