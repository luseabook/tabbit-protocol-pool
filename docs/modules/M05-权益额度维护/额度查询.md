# 额度查询

## 目标

获取账号当前额度、模型权益、重置时间和耗尽状态，供账号池选择器使用。

## 输出

~~~ts
type RefreshQuotaResult = {
  accountId: string;
  quotaState: QuotaState[];
  resetCouponCount?: number;
  accessTier?: string;
  source?: "tabbit-quota-usage" | string;
};
~~~

## 真实协议校准

当前已校准的真实额度查询接口：

~~~text
GET https://web.tabbit.ai/api/commerce/quota/v1/usage?user_id=<user_id>
~~~

请求依赖已登录 Cookie，并携带浏览器风格 `unique-uuid` header；本次抓包未观察到 `x-signature` / `x-nonce`，因此 `ProtocolTabbitClient.refreshQuota()` 不获取 sign key，也不生成签名头。接口必须通过 `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH=/api/commerce/quota/v1/usage` 显式启用。

脱敏响应样例字段：

~~~json
{
  "member_level": "free",
  "usage_percentage": "31.37%",
  "remaining_reset_hours": "161.49",
  "current_cycle_start": "2026.07.03",
  "current_cycle_end": "2026.07.10",
  "unused_reset_coupon_count": 0
}
~~~

归一化规则：

- `member_level` -> `accessTier`。
- `unused_reset_coupon_count` -> `resetCouponCount`。
- `usage_percentage` -> `quotaState[0].usagePercentage`，并以 `>= 100` 判定 `exhausted:true`。
- `current_cycle_end` -> `quotaState[0].resetAt`。
- 由于响应没有总量/剩余额度，`remaining` 和 `limit` 均为 `null`，`unit` 固定为 `usage_percentage`，`source` 固定为 `tabbit-quota-usage`。

## 规则

- 额度未知时不要默认视为耗尽；标为 `unknown` 并降低选择分数。
- 明确耗尽时将账号转为 `quota_exhausted`。
- 查询失败不覆盖上一次成功额度，除非错误证明登录失效。

## 待还原点

- 签到/活动 Pro/重置券是否会影响该 usage 响应。
- usage 百分比与具体模型额度的换算关系。
- `remaining_reset_hours` 是否应转换为稳定 reset 时间，当前先保留 `current_cycle_end`。

## 测试

- [x] quota fixture 归一化：`normalizeQuotaState()` 补齐 model、remaining、limit、unit、resetAt、exhausted、source。
- [x] 查询失败保留旧状态：`refreshQuota()` 捕获错误并返回 failed action。
- [x] exhausted=true 转账号状态：`refreshQuota()` 将账号标记为 `quota_exhausted`。
- [x] 额度恢复转 active：非耗尽 quota 可把 `quota_exhausted` 账号恢复为 `active`。
- [x] 真实 quota usage 接口路径和单位已还原，缺 path/userId/session 时不触网。
