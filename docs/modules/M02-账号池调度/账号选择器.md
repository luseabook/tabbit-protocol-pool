# 账号选择器

## 目标

在用户请求到来时选择最合适的账号，并保证选择过程可解释、可测试、可回放。

## 输入

~~~ts
type PickAccountInput = {
  model: string;
  requiresPremium?: boolean;
  excludeAccountIds?: string[];
  requestId?: string;
};
~~~

## 选择规则

1. 过滤不可用状态：`disabled`、`provisioning`、`login_expired`、未到期 `cooldown`。
2. 过滤明确额度耗尽账号。
3. 过滤不满足模型权益的账号。
4. 对剩余账号计算分数。
5. 分数相同使用 round-robin 指针。

## 推荐分数

| 信号 | 影响 |
|---|---|
| 最近成功时间近 | 加分 |
| failureStreak 高 | 扣分 |
| 剩余额度高 | 加分 |
| 刚冷却恢复 | 小扣分 |
| 与请求模型权益匹配 | 必需条件 |

## 输出

~~~ts
type PickAccountResult = {
  account: Account;
  reason: string;
  candidates: Array<{ accountId: string; score: number; excludedReason?: string }>;
};
~~~

## 边界条件

- 没有可用账号时返回结构化错误，不随机选择失效账号。
- 同一个请求 fallback 时必须传入 `excludeAccountIds`，避免重复使用失败账号。
- 账号数量为 1 时仍走完整选择逻辑，便于测试。

## 测试

- 多账号 round-robin。
- excludeAccountIds 生效。
- 权益不足账号被过滤。
- 无可用账号返回 `no_available_account`。
