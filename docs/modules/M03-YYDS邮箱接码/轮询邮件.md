# 轮询邮件

## 目标

在注册流程中等待 Tabbit 验证码邮件到达，并在超时、限流、空列表时给出可恢复结果。

## 调用路径

~~~text
GET /v1/messages?address=xxx
GET /v1/messages/{id}?address=xxx
GET /v1/sources/{id}?address=xxx
~~~

## 策略

1. 按 `intervalMs` 轮询邮件列表。
2. 根据 from/subject/time window 过滤候选邮件。
3. 对候选邮件读取详情。
4. 详情无法提取验证码时，按需读取 source。
5. 到达 timeout 后返回 `mail_timeout`。

## 退避

- 收到 `429` 时使用 `Retry-After`。
- 网络错误使用指数退避，但总时间不能超过 `timeoutMs`。
- 空列表不算错误。

## 输出

~~~ts
type WaitForMessageResult = {
  message: MailMessage;
  matchedBy: "subject" | "from" | "body" | "latest";
};
~~~

## 测试

- 空列表后成功。
- 429 后按 Retry-After。
- 多封邮件只选时间窗口内候选。
- 超时返回可读错误。
