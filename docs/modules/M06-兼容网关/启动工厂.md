# 启动工厂

本文件记录 M06 当前已实现的 protocol-pool gateway composition layer。它不新增 HTTP 语义，而是把已有配置、账号 store、账号池、runner、OpenAICompat 和 HTTP server 组合成一个可启动对象。

## 定位

~~~text
loadConfig()
  ↓
JsonAccountStore
  ↓
StoredAccountPool
  ↓
PooledRequestRunner
  ↓
LocalToolLoopRunner
  ↓
OpenAICompat + AnthropicCompat
  ↓
createProtocolPoolServer()
  ↓
gateway.server / gateway.start()
~~~

该层的目标是把“能单测的模块”组装成“能监听端口前的网关实例”。真实 Tabbit 文本 send endpoint 和 session verify endpoint 已由 `ProtocolTabbitClient` 支持，但启动工厂仍只负责把 env/options 传给客户端；它不自行探测协议，也不读取 raw cookie。

## 公开接口

~~~ts
type CreateProtocolPoolGatewayOptions = {
  env?: Record<string, string | undefined>;
  config?: ReturnType<typeof loadConfig>;
  store?: JsonAccountStore;
  accountPool?: StoredAccountPool;
  runner?: PooledRequestRunner;
  toolLoopRunner?: { run(input): Promise<unknown> };
  executeLocalToolUse?: (call: object) => Promise<unknown> | unknown;
  localToolExecutor?: ((call: object) => Promise<unknown> | unknown) | { execute(call: object): Promise<unknown> | unknown };
  compat?: OpenAICompat & AnthropicCompat;
  openAiCompat?: OpenAICompat;
  anthropicCompat?: AnthropicCompat;
  protocolClientFactory?: (account: Account) => {
    sendMessage(input): Promise<unknown>;
    verifySession?(input): Promise<unknown>;
    uploadAttachment?(input): Promise<unknown>;
    refreshQuota?(input): Promise<unknown>;
    getLotteryExplorationMe?(input): Promise<unknown>;
    getNewbieExplorationMe?(input): Promise<unknown>;
    getPlacementResources?(input): Promise<unknown>;
    listRewardCardRecords?(input): Promise<unknown>;
    listLotteryHitRecords?(input): Promise<unknown>;
  };
  protocolClientOptions?: object;
  secretStore?: FileSecretStore;
  hydrateSecrets?: boolean;
  fetch?: typeof fetch;
  modelsProvider?: { listModels(): Promise<unknown[]> } | (() => Promise<unknown[]>);
  health?: object | (() => object | Promise<object>);
  now?: () => number;        // account 状态时间，毫秒
  compatNow?: () => number;  // OpenAI response created/created_at，秒
  idFactory?: (kind: string) => string;
};

async function createProtocolPoolGateway(options): Promise<{
  config: object;
  store: JsonAccountStore;
  secretStore: FileSecretStore;
  accountPool: StoredAccountPool;
  protocolClientFactory: Function;
  runner: PooledRequestRunner;
  openAiCompat: OpenAICompat;
  anthropicCompat: AnthropicCompat;
  compat: object;
  server: http.Server;
  start(options?: { host?: string; port?: number }): Promise<http.Server>;
  close(): Promise<void>;
}>;
~~~

辅助导出：

- `createDefaultProtocolClientFactory(options)`：返回为每个账号创建 `ProtocolTabbitClient` 的 factory。
- `listen(server, { host, port })`：Promise 化 `server.listen()`。
- `closeServer(server)`：Promise 化 `server.close()`，未监听时直接返回。

## 默认组合规则

| 组件 | 默认来源 | 可注入覆盖 |
|---|---|---|
| config | `loadConfig(options.env, options)` | `options.config` |
| store | `new JsonAccountStore({ stateDir: config.stateDir })` | `options.store` |
| secretStore | `new FileSecretStore({ stateDir: config.stateDir })` | `options.secretStore` |
| accountPool | `StoredAccountPool.load({ store, now })` | `options.accountPool` |
| protocolClientFactory | `createDefaultProtocolClientFactory()` 后默认包装 secret hydrate；显式 `config.protocol` 会填入默认 protocol options，包含 send、defaultChatSessionId、reqCtx、model scene、session、attachment、quota usage、只读 commerce path 和 M05 side-effect probe path | `options.protocolClientFactory` / `protocolClientOptions` / `hydrateSecrets:false` |
| runner | `new PooledRequestRunner({ accountPool, protocolClientFactory, retryLimit: config.retryLimit })` 后包装为 `LocalToolLoopRunner({ mode: config.compat.toolLoopMode })` | `options.runner` / `options.toolLoopRunner` / `executeLocalToolUse` / `localToolExecutor` |
| openAiCompat | `new OpenAICompat({ runner, now: compatNow, idFactory, stripClientTools })` | `options.openAiCompat` |
| anthropicCompat | `new AnthropicCompat({ runner, now: compatNow, idFactory, stripClientTools })` | `options.anthropicCompat` |
| compat | 合并 OpenAI 与 Anthropic handler | `options.compat` |
| modelsProvider | 显式协议 env opt-in 时使用 `ProtocolTabbitClient.listModels()`；否则 HTTP server 返回空模型列表；runner 使用原始 provider，公开 `/v1/models` 使用账号池 tier 过滤后的 provider | `options.modelsProvider` |
| health | `createGatewayHealthProvider({ accountPool, startedAt, now })`，输出脱敏账号池摘要 | `options.health` |
| server | `createProtocolPoolServer({ apiKey: config.apiKey, compat, modelsProvider, health })` | `options.server` |

## How to：创建本地 gateway 实例

~~~js
import { createProtocolPoolGateway } from "./src/index.js";

const gateway = await createProtocolPoolGateway({
  env: {
    TABBIT_POOL_STATE_DIR: "E:/tmp/tabbit-protocol-pool-state",
    TABBIT_POOL_API_KEY: "sk-tabbit-local",
    TABBIT_POOL_PROTOCOL_FETCH_TRANSPORT: "powershell",
    TABBIT_POOL_PROTOCOL_BASE_URL: "https://web.tabbit.ai",
    TABBIT_POOL_PROTOCOL_SEND_PATH: "/api/v1/chat/completion",
    TABBIT_POOL_PROTOCOL_CHAT_SESSION_AUTO_CREATE: "true",
    TABBIT_POOL_PROTOCOL_CHAT_SESSION_CREATE_PATH: "/newtab",
    TABBIT_POOL_PROTOCOL_CHAT_SESSION_CREATE_ACTION_ID: "00b19386a3892f62370bef2ffacfbd5b58580fcb2a",
    TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH: "/api/commerce/quota/v1/usage",
    TABBIT_POOL_PROTOCOL_ACTIVITY_LOTTERY_PATH: "/api/commerce/activity/v1/lottery/me",
    TABBIT_POOL_PROTOCOL_NEWBIE_EXPLORATION_PATH: "/api/commerce/activity/v1/newbie-exploration/me",
    TABBIT_POOL_PROTOCOL_PLACEMENT_RESOURCES_PATH: "/api/commerce/placement/v1/resources",
    TABBIT_POOL_PROTOCOL_REWARD_CARD_RECORDS_PATH: "/api/commerce/reward/v1/card-records",
    TABBIT_POOL_PROTOCOL_LOTTERY_HIT_RECORDS_PATH: "/api/commerce/lottery/v1/hit-records",
    TABBIT_POOL_PROTOCOL_SESSION_VERIFY_PATH: "/api/v0/user/base-info"
  }
});

await gateway.start();
console.log(gateway.server.address());
~~~

测试中应注入 `protocolClientFactory`，避免真实 Tabbit 网络请求：

~~~js
const gateway = await createProtocolPoolGateway({
  env: { TABBIT_POOL_STATE_DIR: stateDir },
  protocolClientFactory: (account) => ({
    async sendMessage(input) {
      return {
        ok: true,
        contentBlocks: [{ type: "text", text: "ok" }],
        selectedModel: input.model
      };
    }
  })
});
~~~

## 边界

- 不默认监听公网地址；`loadConfig()` 默认 host 仍为 `127.0.0.1`，CLI 示例也只使用本机监听。
- 不把 `TABBIT_POOL_API_KEY`、cookie、session 或 token 写入文档示例。
- 默认创建 `FileSecretStore` 读取 `cookieJarRef`，但不会把 raw secret 写入账号 JSON。
- 默认 `/health` 只输出账号池统计摘要，不输出邮箱原文、cookie/session/token 或 `cookieJarRef`。
- 不改变 HTTP route adapter 的 JSON/SSE 行为。SSE 仍是后续单独切片。
- 不自动探测真实 Tabbit send endpoint；已校准路径是 `/api/v1/chat/completion`，仍必须通过 `TABBIT_POOL_PROTOCOL_SEND_PATH`、`protocolClientOptions.sendPath` 或自定义 `protocolClientFactory` 提供。
- 不自动绕过 edge 风控；当前 Windows 本地 Node/curl 被 Tabbit edge 403 拦截时，必须显式设置 `TABBIT_POOL_PROTOCOL_FETCH_TRANSPORT=powershell` 或注入自定义 `fetch`。
- 不自动探测真实 Tabbit session verify endpoint；已校准路径是 `/api/v0/user/base-info`，仍必须通过 `TABBIT_POOL_PROTOCOL_SESSION_VERIFY_PATH`、`protocolClientOptions.sessionVerifyPath` 或自定义 `protocolClientFactory` 提供。
- 不自动探测未知会话协议；已校准的真实 chat completion 会在缺少显式会话 id 时，通过 `TABBIT_POOL_PROTOCOL_CHAT_SESSION_AUTO_CREATE` 配置的 `/newtab` Next Server Action 自动创建会话。显式 `TABBIT_POOL_PROTOCOL_CHAT_SESSION_ID`、账号元数据中的 chat session id，或单次 `sendMessage({ chatSessionId })` 仍优先使用。账号元数据中的 `chatSessionId` 可通过 `accounts import-session --chat-session-id <id>` 从已登录浏览器会话导入，gateway 输出不会打印该值。
- 不自动探测真实 Tabbit attachment upload endpoint；必须通过 `TABBIT_POOL_PROTOCOL_ATTACHMENT_UPLOAD_PATH`、`TABBIT_POOL_PROTOCOL_ATTACHMENT_COMPLETE_UPLOAD_PATH`、`protocolClientOptions.attachmentUploadPath` / `attachmentCompleteUploadPath` 或自定义 `protocolClientFactory` 提供。真实 `sendMessage({ attachments })` 支持已上传附件引用；同时配置 presign/complete path 时，raw/base64 附件会先上传再映射为消息 `references`。
- 不自动探测真实 Tabbit quota usage endpoint；已校准路径是 `/api/commerce/quota/v1/usage`，仍必须通过 `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH`、`protocolClientOptions.quotaUsagePath` 或自定义 `protocolClientFactory` 提供。该路径只支持已登录 Cookie + `userId` 查询；它是默认维护链唯一自动接入的真实权益接口。
- 不自动探测真实 Tabbit activity/newbie/placement/reward/lottery/M05 side-effect endpoint；已校准的只读路径和 side-effect probe 路径必须通过对应 `TABBIT_POOL_PROTOCOL_*_PATH`、`protocolClientOptions` 或自定义 `protocolClientFactory` 提供。gateway 只负责把这些 path 传给 `ProtocolTabbitClient`，不会把只读状态/资源查询或 POST 型副作用接入 `maintain`；POST 型方法还必须由调用方传入 `confirmSideEffect:true`。
- 不默认请求 Tabbit 模型目录；只有显式协议 env opt-in 或注入 `modelsProvider` 时，`/v1/models` 才会读取上游模型目录。原始 provider 会传给默认 `PooledRequestRunner`，用于根据 `requires_premium` / `model_access_type` 做 paid 账号路由：`pro`、`paid_*`、`premium_only` 与 `Claude-Opus-*` 均按本地最高可见 `pro` tier 处理；公开 `/v1/models` 会隐藏当前账号池没有可选账号承接的模型；已知不可用的 `Claude-Opus-4.7` 会在目录输出和请求入口被排除。
- 不默认剥离客户端工具；只有显式 `TABBIT_POOL_COMPAT_STRIP_CLIENT_TOOLS=true` 时，OpenAI/Anthropic handler 才会为文本链路验收剥离已知客户端内置协作工具。
- 不默认执行本地工具；`TABBIT_POOL_TOOL_LOOP_MODE=local_executes_tools` 仍必须由宿主注入 `executeLocalToolUse` 或 `localToolExecutor.execute`。默认不内置 shell、web、js、fetch 等工具，且真实 `/api/v1/chat/completion` 原生工具字段仍会被剥离后再调用。本地执行可通过 `TABBIT_POOL_LOCAL_TOOL_ALLOWLIST`、`TABBIT_POOL_LOCAL_TOOL_MAX_ROUNDS`、`TABBIT_POOL_LOCAL_TOOL_TIMEOUT_MS`、`TABBIT_POOL_LOCAL_TOOL_MAX_RESULT_CHARS` 设置 allowlist、轮数、单工具超时和输出截断。

## 测试契约

`test/protocol-pool-gateway.test.js` 覆盖：

1. 从 `stateDir/accounts.json` 加载账号。
2. Chat Completions 请求经 HTTP server → OpenAICompat → runner → StoredAccountPool → protocol client factory 完成。
3. 请求成功后 `StoredAccountPool` 把 `lastSuccessAt` 与 audit 写回 JSON store。
4. `modelsProvider` 可从启动工厂透传到 `/v1/models`。
5. `gateway.start({ port: 0 })` 默认绑定 `127.0.0.1`，`gateway.close()` 能关闭 server。
6. `tabbit-pool serve/start` 通过注入 gatewayFactory 验证 host/port 传递、shutdown 后 close，以及非法 port 不创建 gateway。
7. `cookieJarRef` 会通过 `FileSecretStore` hydrate 到运行时账号对象，且不写回 `accounts.json`。
8. `/v1/messages` 通过 AnthropicCompat 接入同一条 pooled runner 链路。
9. 默认 `/health` 包含账号池统计摘要，且不泄露账号 secret 字段。
10. 显式 `TABBIT_POOL_PROTOCOL_FETCH_TRANSPORT`、`TABBIT_POOL_PROTOCOL_SEND_PATH`、`TABBIT_POOL_PROTOCOL_CHAT_SESSION_ID`、`TABBIT_POOL_PROTOCOL_CHAT_SESSION_CREATE_PATH`、`TABBIT_POOL_PROTOCOL_CHAT_SESSION_CREATE_ACTION_ID`、`TABBIT_POOL_PROTOCOL_CHAT_SESSION_AUTO_CREATE` 和 `TABBIT_POOL_PROTOCOL_REQ_CTX` 会传给默认 `ProtocolTabbitClient`，请求仍使用本地 secret hydrate 后的 `Cookie`；显式注入的 `fetch` 优先于 env transport。
11. 显式 `TABBIT_POOL_PROTOCOL_MODEL_CATALOG_PATH` 会让 `/v1/models` 使用 `ProtocolTabbitClient.listModels()`，默认带 `scene=chat`，并保留 `options.modelsProvider` 的覆盖优先级；paid 模型目录也会参与默认 runner 的账号选择。
12. 显式 `TABBIT_POOL_PROTOCOL_ATTACHMENT_UPLOAD_PATH` / `TABBIT_POOL_PROTOCOL_ATTACHMENT_COMPLETE_UPLOAD_PATH` 会传给默认 `ProtocolTabbitClient`；secret-hydrating wrapper 会先读取 `cookieJarRef`，再把 hydrated `Cookie` 传给上传请求和 send 前 raw/base64 自动上传。
13. 显式 `TABBIT_POOL_PROTOCOL_QUOTA_USAGE_PATH` 会传给默认 `ProtocolTabbitClient.refreshQuota()`；secret-hydrating wrapper 会先读取 `cookieJarRef`，再把 hydrated `Cookie` 传给 quota usage 请求。
14. 显式 `TABBIT_POOL_PROTOCOL_SESSION_VERIFY_PATH` 会传给默认 `ProtocolTabbitClient.verifySession()`；secret-hydrating wrapper 会先读取 `cookieJarRef`，再把 hydrated `Cookie` 同时作为 account cookie 和 session 传给验证请求。
15. 显式 `TABBIT_POOL_PROTOCOL_ACTIVITY_LOTTERY_PATH`、`TABBIT_POOL_PROTOCOL_NEWBIE_EXPLORATION_PATH`、`TABBIT_POOL_PROTOCOL_PLACEMENT_RESOURCES_PATH`、`TABBIT_POOL_PROTOCOL_REWARD_CARD_RECORDS_PATH` 和 `TABBIT_POOL_PROTOCOL_LOTTERY_HIT_RECORDS_PATH` 会传给默认 `ProtocolTabbitClient`；secret-hydrating wrapper 会在只读 commerce 查询前读取 `cookieJarRef` 并传入 hydrated `Cookie`。
16. 显式 `TABBIT_POOL_COMPAT_STRIP_CLIENT_TOOLS` 会传给 OpenAI/Anthropic handler，默认关闭。
17. 显式 `TABBIT_POOL_TOOL_LOOP_MODE=local_executes_tools` 会通过 LocalToolLoopRunner 包装 pooled runner；只有注入 executor 时才执行工具，并确保协议客户端收到的 `tools/toolChoice/parallelToolCalls` 为 `null`。
18. 本地工具 loop guardrails 会从 env 进入 `config.compat.localToolLoop` 并传给 LocalToolLoopRunner；Chat Completions、Responses 和 Anthropic Messages 三入口共用 allowlist、最大轮数、单工具超时和结果截断策略。
