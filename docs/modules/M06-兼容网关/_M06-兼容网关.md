# M06-兼容网关

## 定位

CompatGateway 对外保持 OpenAI/Anthropic 兼容接口，对内接入 PooledRequestRunner、AccountPool 和 ProtocolTabbitClient。外部客户端不应感知账号池细节。

M06 当前拆成两层：

1. OpenAICompat 纯 handler：已实现，接收 JSON body，返回 `{ status, body }`，`stream:true` 时可附带非公开 `stream.deltas`。
2. AnthropicCompat 纯 handler：已实现 Messages JSON，接收 JSON body，返回 `{ status, body }`，`stream:true` 时可附带非公开 `stream.deltas`。
3. HTTP Route Adapter：已实现 JSON 骨架、OpenAI/Anthropic SSE adapter、async iterable chunked flush、route-specific SSE error frame 与客户端断开后的 async iterator cancellation，负责本地 HTTP 监听、认证、JSON 解析、响应写回和成功结果 SSE framing。
4. Gateway 启动工厂：已实现，负责把 config、账号元数据 store、StoredAccountPool、runner、OpenAICompat 与 HTTP server 组合成可启动对象。

工具调用当前包含“基础兼容通道”和“受控本地 loop”两层：OpenAI Chat/Responses 的非空 `tools`、`tool_choice`、`parallel_tool_calls` 与 Anthropic Messages 的非空 `tools`、`tool_choice` 默认从 compat 透传到 PooledRequestRunner；没有真实 tools 时，compat 会忽略 `auto/none` 这类 no-op 工具选择和孤立 `parallel_tool_calls`，避免纯文本请求误触发真实工具未支持错误。旧显式 sendPath 骨架会写入 ProtocolTabbitClient signed send body，真实 `/api/v1/chat/completion` 分支返回 `unsupported_feature/TOOL_FIELDS_UNSUPPORTED`。协议客户端可把上游 OpenAI `tool_calls` / buffered stream `tool_calls` / Anthropic `tool_use` / buffered Anthropic `input_json_delta` 归一化为内部 `tool_use` block；compat 会输出 OpenAI Chat `message.tool_calls`、Responses `function_call` output item 与 Anthropic `tool_use` content block；Chat SSE 会输出 `tool_calls` delta，Responses SSE 会输出 function call item events；客户端后续提交的 OpenAI Chat `role:"tool"` / `tool_call_id`、Responses `function_call_output`、Anthropic `tool_result` 等工具回合输入也会在 compat/runner 层保留。`TABBIT_POOL_TOOL_LOOP_MODE=local_executes_tools` 且宿主注入 executor 时，LocalToolLoopRunner 会剥离真实协议不支持的原生工具字段、解析 JSON tool_use 文本并执行注入工具；默认不启用且不内置 shell/web/js/fetch 工具。async 上游 OpenAI/Anthropic 工具 delta 已接入本地 chunked SSE adapter。

## 兼容范围

第一阶段建议继承 Tabbit2API 已有能力中的核心子集：

- GET /health
- GET /v1/models
- POST /v1/responses
- POST /v1/chat/completions
- POST /v1/messages（Anthropic，已实现 JSON 与 SSE adapter）

Assistants、Threads、Realtime 可在协议直连稳定后再迁移。

## 请求链路

~~~text
HTTP route adapter
  ↓ auth + JSON parse
OpenAICompat / AnthropicCompat
  ↓ normalize request
PooledRequestRunner
  ↓ pick account
ProtocolTabbitClient
  ↓ send protocol request
OpenAICompat / AnthropicCompat
  ↓ map response/error
HTTP route adapter
  ↓ write compatible output
client
~~~

## 错误映射

| 内部错误 | OpenAI 风格 | Anthropic 风格 |
|---|---|---|
| no_available_account | 503 api_error | 503 api_error |
| login_required | 401 authentication_error | 401 authentication_error |
| invalid_request | 400 invalid_request_error | 400 invalid_request_error |
| quota_exhausted | 503 api_error | 503 api_error |
| protocol_changed | 502 api_error | 502 api_error |
| timeout | 504 api_error | 504 api_error |
| unsupported_feature | 502 api_error 或 400 invalid_request_error | 400 invalid_request_error |

HTTP 层新增错误：

| 场景 | 状态码 | OpenAI type | code |
|---|---:|---|---|
| 缺少或错误 API key | 401 | authentication_error | invalid_api_key |
| 非 JSON / 坏 JSON | 400 | invalid_request_error | invalid_json |
| 未知路由 | 404 | invalid_request_error | not_found |

## 模型策略

- 保留 tabbit/priority。
- 对客户端公开统一模型名。
- ProtocolTabbitClient.normalizeModelCatalog 已能把上游模型转为 tabbit/... 形式。
- /v1/models 路由应把 selectedModel 映射为 tabbit_selected_model，不暴露账号 ID。
- 在响应 metadata 或调试日志中记录 attempted account/model，但默认不暴露账号敏感信息。

## 流式输出

完整端到端设计见 [流式 SSE 链路](../../10-流式SSE链路.md)。

- OpenAI Chat Completions 已实现 SSE adapter：成功 JSON 转换为 role delta、一个或多个 content delta、可选 `tool_calls` delta、finish chunk 与 `[DONE]`。
- OpenAI Responses 已实现 SSE adapter：成功 JSON 转换为 `response.created`、一个或多个 `response.output_text.delta`、`response.completed` 与 `[DONE]`。
- handler 提供数组 `stream.deltas` 时保留上游解析出的文本分片；handler 提供 async iterable `stream.deltas` 时 HTTP 层 chunked flush，字符串按文本输出，`tool_call_delta` 按 Chat/Responses/Anthropic 工具事件输出；没有 `stream.deltas` 时使用完整文本 fallback。`ProtocolTabbitClient` 在显式 `sendPath`、2xx、支持的 SSE/NDJSON Content-Type 且 response.body 可读时，已能直接产出 async iterable `streamDeltas`。
- 非 2xx handler 结果保持 JSON error，不启动 SSE。
- Anthropic Messages 已实现 SSE adapter：成功 JSON 转换为 `message_start`、content block events、`message_delta`、`message_stop`；`tool_use` content block 会按 Anthropic 事件输出。
- async iterable 迭代期错误发生在 SSE headers 已发送之后，HTTP 层不会改写 JSON：Chat 输出 OpenAI `data:error` + `[DONE]`，Responses 输出 `event: response.failed` + `[DONE]`，Anthropic 输出 `event:error`。
- 下游客户端断开时，HTTP writer 会 abort streaming adapter 的 close signal，并请求 async `stream.deltas` iterator `return()`；该行为覆盖可取消 async source，真实上游 fetch/body 取消时机仍由协议客户端和真实 fixture 校准。

已覆盖：

- OpenAI Chat Completions 请求归一化。
- OpenAI Responses 输入归一化。
- OpenAI JSON 成功响应 shape。
- OpenAI 错误映射。
- PooledRequestRunner 成功、失败、fallback、无账号。
- /health。
- /v1/models OpenAI shape。
- /v1/chat/completions HTTP auth、JSON parse、handler wiring。
- /v1/responses HTTP auth、JSON parse、handler wiring。
- /v1/messages HTTP auth、JSON parse、Anthropic handler wiring。
- OpenAI Chat/Responses `stream:true` SSE adapter，包含完整文本 fallback、数组 `stream.deltas` 分片保留、async iterable 文本 chunked flush 和 async 工具调用事件 flush。
- Anthropic Messages `stream:true` SSE adapter，包含完整文本 fallback、数组 `stream.deltas` 分片保留、async iterable 文本 chunked flush 和 async `tool_use` / `input_json_delta` flush。
- async iterable `stream.deltas` 在第一段 delta flush 后 reject 时，三类路由输出各自兼容的 SSE error frame，且不会触发 headers sent 后 JSON 写回。
- `/v1/chat/completions` async SSE 在下游客户端读取第一段 delta 后断开时，会请求 `stream.deltas` iterator `return()`，避免继续消费可取消的 delta source。
- OpenAI/Anthropic `stream:true` 非 2xx handler 错误保持 JSON。
- 坏 JSON 400、错误 API key 401、未知路由 404。
- Gateway 工厂加载 `stateDir/accounts.json`、组合请求链路、成功后写回账号状态。
- Gateway 工厂透传 `modelsProvider`，`start({ port:0 })` 默认绑定 `127.0.0.1` 并可关闭。
- ProtocolTabbitClient response.body async delta producer，经 OpenAI/Anthropic compat 非公开 `stream.deltas` 元数据接入 HTTP async SSE flush。
- OpenAI Chat/Responses 与 Anthropic Messages 官方工具字段从 compat -> runner 的透传；旧显式 sendPath 会继续写入 ProtocolTabbitClient signed body，真实 send 分支明确拒绝原生工具字段。
- ProtocolTabbitClient 上游 OpenAI `tool_calls` / buffered stream `tool_calls` / async stream `tool_calls` / Anthropic `tool_use` / buffered 或 async Anthropic `input_json_delta` 响应归一化，OpenAI Chat `tool_calls`、Responses `function_call` output item、Anthropic `tool_use` content block、Chat SSE `tool_calls` delta、Responses function_call SSE item events 与 Anthropic async `tool_use` SSE 输出。
- OpenAI Chat `tool_calls` / `tool_call_id`、OpenAI Responses `function_call` / `function_call_output`、Anthropic `tool_use` / `tool_result` 工具回合输入在 compat、runner 和旧显式 sendPath signed body 中保持结构。
- LocalToolLoopRunner 在 `local_executes_tools` 模式下剥离真实协议不支持的工具字段、执行注入 executor、追加 tool result 并受最大轮数限制；默认 `client_executes_tools_first` 保持透传，`disabled` 剥离工具字段。
