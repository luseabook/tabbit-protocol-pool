# 02-架构文档

## 总体架构

~~~text
OpenAI / Anthropic Client
        ↓
HTTP Route Adapter（JSON + SSE 已实现）
        ↓
OpenAICompat / AnthropicCompat
        ↓
Pooled Request Runner
        ↓
Account Pool Scheduler
        ↓
Protocol Tabbit Client
        ↓
Tabbit Web HTTP / Stream APIs
~~~

辅助维护链路：

~~~text
Scheduler / CLI
   ├─ Benefits & Quota Maintainer
   ├─ Account Provisioner
   └─ YYDS Mail Provider
~~~

当前已实现到配置/脱敏、YYDSMailProvider、ProtocolTabbitClient、AccountPool、JsonAccountStore/StoredAccountPool、PooledRequestRunner、受控 LocalToolLoopRunner、OpenAICompat、AnthropicCompat、OpenAI/Anthropic 官方工具字段透传通道、基础 tool call 输出映射、工具回合输入保真、buffered OpenAI stream `tool_calls` 聚合、buffered Anthropic stream `tool_use` / `input_json_delta` 聚合、Responses `function_call` SSE item events、async 上游工具 delta 转本地 SSE、HTTP Route Adapter JSON/SSE 骨架、HTTP async SSE flush、协议 response.body async delta producer、stream error frame 基础分类传播。真实 Tabbit 消息 send endpoint、session verify 路径、真实签名 fixture、已上传附件引用结构和 raw/base64 附件 COS 自动上传已完成校准；真实上游私有工具语义仍在后续校准阶段。LocalToolLoopRunner 仅在显式 `local_executes_tools` 且宿主注入 executor 时执行本地工具，默认不启用且不内置工具。

## 技术栈建议

- Runtime：Node.js ESM。
- HTTP Server：优先复用现有 Tabbit2API 的原生 node:http 兼容层；如独立项目复杂度上升，再评估 Fastify。
- HTTP Client：Node fetch + cookie jar。
- Storage：当前实现 JSON 账号元数据 store；后续可评估 SQLite 与系统密钥链。`AccountPool` 是内存状态机，`StoredAccountPool` 负责状态写回。
- Testing：Node 内置测试 node --test。
- Secret：环境变量 + 本地加密/权限限制文件。

## 核心模块

### 1. ProtocolTabbitClient

职责：维护 cookie jar、获取 sign key、生成签名头、发送 chat 请求、解析响应流、查询模型目录。详见 [M01-Tabbit协议客户端](modules/M01-Tabbit协议客户端/_M01-Tabbit协议客户端.md)。

### 2. AccountPool

职责：加载账号列表、选择可用账号、记录账号健康状态、根据错误更新冷却/额度/登录状态；当前通过 JsonAccountStore/StoredAccountPool 持久化账号元数据。详见 [M02-账号池调度](modules/M02-账号池调度/_M02-账号池调度.md) 与 [账号元数据持久化](modules/M02-账号池调度/账号元数据持久化.md)。

### 3. YYDSMailProvider

职责：创建临时邮箱、读取邮件列表、读取邮件详情和源码、从邮件中提取验证码。详见 [M03-YYDS邮箱接码](modules/M03-YYDS邮箱接码/_M03-YYDS邮箱接码.md)。

### 4. AccountProvisioner

职责：生成注册邮箱、驱动 Tabbit 注册协议、等待邮箱验证码、完成登录并保存 session、触发首次权益领取。详见 [M04-账号注册初始化](modules/M04-账号注册初始化/_M04-账号注册初始化.md)。

### 5. BenefitsMaintainer

职责：每日签到、领取活动 Pro、查询额度和重置券、使用重置券、更新账号状态。详见 [M05-权益额度维护](modules/M05-权益额度维护/_M05-权益额度维护.md)。

### 6. CompatGateway

职责：对外提供 OpenAI/Anthropic 兼容 API，把客户端模型名映射到 Tabbit 模型，把 Tabbit 错误翻译成兼容错误。当前拆成两层：

- OpenAICompat：已实现的纯 handler，详见 [OpenAI Chat/Responses 处理器](modules/M06-兼容网关/OpenAI-Chat-Responses处理器.md)。
- HTTP Route Adapter：已实现的 node:http JSON 路由层和 OpenAI/Anthropic SSE adapter，详见 [HTTP 路由层](modules/M06-兼容网关/HTTP路由层.md) 与 [流式 SSE 链路](10-流式SSE链路.md)。

M06 总览见 [M06-兼容网关](modules/M06-兼容网关/_M06-兼容网关.md)。

## 数据流：一次模型调用

~~~text
1. 客户端 POST /v1/chat/completions 或 /v1/responses
2. HTTP Route Adapter 校验本地 API key，解析 JSON body
3. OpenAICompat 归一化消息/附件/模型参数，保留官方工具字段和工具回合输入结构
4. PooledRequestRunner 请求 AccountPool 选择 active 账号
5. ProtocolTabbitClient 生成签名头
6. ProtocolTabbitClient 请求 Tabbit 消息接口
7. 解析返回文本/流
8. 成功：OpenAICompat 构造兼容响应
9. 失败：分类错误，更新账号状态，必要时重试其他账号
10. HTTP Route Adapter 写回 OpenAI/Anthropic 风格 JSON 或 SSE
~~~

## 数据流：注册账号

~~~text
1. AccountProvisioner 请求 YYDSMailProvider.createInbox()
2. 调用 Tabbit 发送验证码接口（待还原）
3. YYDSMailProvider.waitForVerificationCode()
4. 调用 Tabbit 注册/验证接口（待还原）
5. 调用 Tabbit 登录/会话初始化接口（待还原）
6. 保存 cookie/session
7. BenefitsMaintainer.claimProIfAvailable()
8. AccountPool 写入 active 账号
~~~

## 数据流：每日维护

~~~text
1. Scheduler 扫描账号池
2. 对 active/cooldown/quota_exhausted/login_expired 分策略处理
3. 签到
4. 领取可用权益
5. 查询额度与重置券
6. 若额度耗尽且有券，按策略使用重置券
7. 写回账号状态与下一次维护时间
~~~

## 安全与维护约束

- YYDS_MAIL_API_KEY 不写入文档和代码。
- 账号 cookie/session 不进入 Git。
- 所有本地状态目录默认应加入 .gitignore。
- 日志默认脱敏 cookie、token、email localPart。
- 协议接口变更时，通过 contract tests 快速定位。
- HTTP server 默认监听 127.0.0.1，不新增公网部署路径。
