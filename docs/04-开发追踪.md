## 剩余工作总账

当前默认状态目录若缺少账号、fixture 或 `readiness.json`，`readiness` / `fixtures audit` 返回 `blocked` 是安全默认。复用本机外部脱敏 evidence 时，先设置 `TABBIT_POOL_STATE_DIR`，再运行 `tabbit-pool readiness doctor --json`、`tabbit-pool readiness --json` 和 `tabbit-pool fixtures audit --json` 判断 core gateway/chat 缺口；`readiness doctor` 的 `calibrationBacklog` 会同步显示 auth/benefits/session/upstream 扩展校准缺口。

| 优先级 | 工作项 | 当前状态 | 下一步验收 |

## 每次实现前检查

1. 找到对应模块文档。
2. 确认输入、输出、状态变更和失败分类。
3. 先补测试或 fixture，再写实现。
4. 修改公开 API、配置或数据结构后，同步更新本目录文档。
5. 修改 src/index.js 导出后，同步更新 [09-实现接口参考](09-实现接口参考.md)。
6. 实现 HTTP server 前，先执行 [HTTP server 实施计划](plans/2026-07-02-http-server-foundation.md) 的 TDD 清单；扩展 OpenAI `stream:true` 时先读 [OpenAI SSE fallback 计划](plans/2026-07-02-openai-sse-fallback.md)，扩展 Anthropic `stream:true` 时先读 [Anthropic SSE fallback 计划](plans/2026-07-02-anthropic-sse-fallback.md)。

8. 扩展 M01 附件上传或消息附件引用时先读 [Protocol Attachment Upload Skeleton 计划](plans/2026-07-02-protocol-attachment-upload-skeleton.md)，并确认本次改动是“已上传附件引用”还是“raw/base64 自动上传后发送”，不要把上传 endpoint 骨架等同于完整附件闭环。
