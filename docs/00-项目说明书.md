# 00-项目说明书

## 项目名称

Tabbit Protocol Pool（暂定名）

## 一句话说明

把 Tabbit Web Chat 从“Playwright 驱动浏览器 UI”改造成“协议直连 + 多账号池 + 自动维护”的本地兼容网关。

## 背景

当前 Tabbit2API 依赖本机 Tabbit 客户端登录态和浏览器 UI。这个方案能快速跑通本地兼容网关，但存在三个扩展瓶颈：

1. 运行时必须打开或复用 Tabbit 浏览器环境。
2. 单账号额度不足时，外部客户端只能看到一次失败，不能自动切换账号。
3. 注册、签到、权益领取、额度刷新仍靠人工维护。

用户希望先完成文档，再以新项目目录开始实现，核心方向是完全脱离浏览器 UI、支持账号池轮询、自动接码注册和每日额度维护。

## 目标用户

- 本地自用的 AI 工具用户。
- 需要把 Tabbit 模型能力接入 Codex、Claude Code、OpenClaw、Hermes Agent、OpenAI SDK、Anthropic SDK 的开发者。
- 需要自动维护账号额度、签到、权益状态的本地网关运维者。

## 核心目标

1. **脱离浏览器 UI**：运行时不再依赖 Playwright 页面操作发送消息。
2. **协议直连**：还原 Tabbit Web 内部 HTTP/流式协议、签名头、cookie/session 使用方式。
3. **账号池轮询**：多个账号按健康度、额度、权限、冷却状态调度。
4. **自动接码**：接入 YYDS Mail/215 邮箱 API，自动创建邮箱并读取验证码。
5. **账号生命周期自动化**：注册、登录、保存 session、领取活动 Pro、高级模型权益初始化。
6. **额度维护自动化**：每日签到、查询额度、领取赠送额度、使用重置券、刷新账号状态。
7. **兼容网关**：对外继续暴露 OpenAI / Anthropic 兼容接口，并优先保留 Codex、Claude Code 等编码工具依赖的官方工具字段。

## 非目标

- 不改造 Tabbit 官方服务端。
- 不把本地服务默认暴露公网。
- 不在代码或文档中保存真实密钥、真实账号密码、真实 cookie。
- 初期不做 Web 管理后台，先以配置文件、CLI、日志满足可验证闭环。
- 初期不做分布式账号池，也不承诺跨机器同步 session。

## 当前信息来源

- Tabbit 模型目录接口：`https://web.tabbit.ai/proxy/v1/model_config/models?a=0&scene=chat`。
- Tabbit Chat 页面线索：`https://web.tabbit.ai/session/<session-id>` 和 `https://web.tabbit.ai/chat/new`。
- Tabbit 签名 key 接口：`https://web.tabbit.ai/chat/sign-key`。
- Tabbit 文本发送接口：`POST https://web.tabbit.ai/api/v1/chat/completion`。
- Tabbit session verify 接口：`GET https://web.tabbit.ai/api/v0/user/base-info`。
- YYDS Mail API 摘要：`https://vip.215.im/v1/llms.txt`。
- YYDS Mail API 文档：`https://vip.215.im/docs`。
- 现有 Tabbit2API 源码：OpenAI/Anthropic 兼容层、模型别名、错误输出和 SSE 行为。

Tabbit 文本发送、session verify、签名、模型目录、已上传附件引用结构和 raw/base64 附件 COS 自动上传已按本地抓包与前端代码完成校准；注册/登录、未验证权益副作用和真实上游私有工具语义仍以“待还原”或“待验证”标记。官方原生工具字段已确认不被真实发送接口支持。

## 成功标准

- 不打开 Tabbit 浏览器 UI，也能通过协议请求完成一次模型回复。
- 至少两个账号可被轮询调度。
- 账号额度耗尽时能自动切换下一个可用账号。
- 可自动从 YYDS Mail 邮箱读取注册验证码。
- 每日维护任务能刷新签到、权益、额度、重置券状态。
- 外部客户端无需感知账号池，仍使用本地 OpenAI/Anthropic 兼容 API。
- OpenAI / Anthropic 风格请求中的工具定义字段不会被兼容层吞掉，可透传到协议请求体；基础 tool call 输出可映射为官方兼容结构；受控本地工具 loop 已有 opt-in wrapper，默认不启用且不内置任何工具。
- 测试中能用录制样本验证签名、错误分类、账号选择和兼容层响应。
