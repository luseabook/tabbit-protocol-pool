# 05-术语表

| 术语 | 定义 |
|---|---|
| 协议直连 | 不通过浏览器 UI 操作，而是用 HTTP/流式协议直接调用服务端接口。 |
| 账号池 | 多个 Tabbit 账号 session/cookie 的集合。 |
| 账号状态机 | 描述账号从 active 到 quota_exhausted/login_expired/cooldown 等状态的转换规则。 |
| YYDS Mail | `vip.215.im` / `maliapi.215.im` 提供的临时邮箱 API 服务。 |
| 接码 | 通过邮箱 API 获取注册验证码。 |
| sign key | Tabbit `/chat/sign-key` 返回的签名密钥材料；真实文本发送使用它计算 `x-nonce`，缓存和账号关联仍需长期观察。 |
| 签名头 | `x-timestamp`、`x-nonce`、`x-signature` 等请求头；真实文本发送中 `x-signature` 是 UUID/randomUUID 风格字符串，`x-nonce = HMAC-SHA256(signKey, timestamp.signature.sha256(bodyText))`。 |
| 重置券 | 用户描述的可用于重置用量/额度的券，具体接口待还原。 |
| 兼容网关 | 对外模拟 OpenAI/Anthropic API 的本地服务。 |
| HTTP Route Adapter | 兼容网关的 HTTP 层，负责认证、路由、JSON 解析和响应写回。 |
| OpenAICompat | 当前已实现的 OpenAI Chat/Responses 纯 handler，不直接监听端口。 |
| PooledRequestRunner | 串联账号选择、协议发送、失败记录和账号 fallback 的请求执行器。 |
| fixture | 脱敏后的请求/响应样本，用于回归测试协议解析和错误分类。 |
| Retry-After | HTTP 响应头，常见于 `429`，表示下一次请求前应等待的时间。 |
| temp token | YYDS Mail 临时 inbox 流程中的短期 Bearer token，不等同于用户 API key。 |
